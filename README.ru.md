# Claude-Codex Review Skill

Два ИИ-агента итеративно ревьюируют планы и код — Claude Code как инициатор, Codex как независимый ревьюер — находя проблемы, которые один агент пропускает.

## Зачем dual review

Один ИИ-агент склонен быть последовательным с самим собой: он делает одни и те же допущения на протяжении всей задачи и редко ставит под сомнение собственные решения. Второй, независимый агент ломает этот паттерн:

- **Слепые зоны** — у каждого агента свои предубеждения; перекрёстная проверка выявляет проблемы, которые ни один из них не нашёл бы в одиночку
- **Дисциплина доказательств** — протокол требует, чтобы каждое замечание ссылалось на конкретный файл, метод или цепочку логики, отсеивая расплывчатые «может быть»
- **Итеративная сходимость** — принятые замечания исправляются и перепроверяются, отклонённые требуют явной аргументации, поэтому каждый раунд даёт измеримый прогресс
- **Аудиторский след** — каждый раунд это markdown-файл в `.dual-review/`, формирующий полную историю решений для команды

## Окружение

Этот скилл оптимизирован и отлажен для **Claude Code и Codex, работающих как расширения VS Code** в одном воркспейсе. Оба агента имеют доступ к каталогу проекта и работают с файлами напрямую.

Это не MCP-интеграция с Codex и не CLI-режим — это файловый протокол между двумя панелями расширений VS Code.

## Как это работает

Сессия ревью живёт в `.dual-review/<session_id>/` и проходит через раунды:

```
R1-01-round-start.md     <- Claude готовит контекст ревью
R1-02-codex-claimed.flg  <- Codex забирает раунд
R1-03-codex-review.md    <- Codex пишет структурированное ревью
R1-04-claude-claimed.flg <- Claude обрабатывает результат
```

В каждом раунде Claude анализирует обратную связь — принимает, отклоняет или исправляет каждое замечание с явной аргументацией. Если вердикт `NEEDS_WORK`, Claude вносит изменения и открывает новый раунд. Цикл продолжается до `APPROVED`, `REJECTED` или достижения `max_rounds`, после чего создаётся `final.md` с итогом сессии.

### Области ревью

| Область | Когда использовать |
|---|---|
| `plan-only` | Ревью плана до изменений в коде |
| `production-change` | Ревью уже внесённых изменений |
| `architecture-check` | Оценка архитектурного решения |
| `lookup-test` | Проверка гипотезы или поведения API |

### Контракт качества

Каждое ревью требует:
- Только значимые замечания — без стилистических придирок и слабых гипотез
- Каждое замечание подтверждено доказательством (путь к файлу, имя метода, логика)
- Отдельная проверка на пропущенные требования, риски регрессии и over-engineering

## Установка

Скопируйте каталог `claude-code/` в скиллы вашего проекта:

```bash
mkdir -p .claude/skills/codex-dual-review-file-based
cp claude-code/* .claude/skills/codex-dual-review-file-based/
```

## Использование

В Claude Code:

```
/codex-dual-review-file-based <описание задачи> [путь/к/плану.md] [max_rounds=5]
```

Claude подготовит первый раунд и предложит передать инструкции ревьюера в Codex. Фоновый вотчер автоматически обнаружит завершение ревью и запустит следующий шаг.

## Структура файлов

```
claude-code/
  SKILL.md              # Протокол инициатора (определение скилла Claude Code)
  reviewer-prompt.txt   # Протокол ревьюера (промпт для Codex)
  wait-for-review.sh    # Скрипт фонового ожидания
docs/
  review-findings.md    # Self-review протокола (проектные решения, краевые случаи)
```

## Требования

- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) как расширение VS Code
- [Codex](https://openai.com/index/codex/) как расширение VS Code (в том же воркспейсе)
- Bash (для `wait-for-review.sh`)

## Ограничения

- Codex должен иметь доступ на чтение и запись в каталог `.dual-review/` проекта
- Вотчер `wait-for-review.sh` опрашивает каждые 10 секунд — не мгновенно
- Файлы сессии append-only по дизайну — ни один файл не перезаписывается и не удаляется в рамках сессии
